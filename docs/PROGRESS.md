### ✅ Импорт: чиним ПРИЧИНУ (фидбек: «делай корректный импорт, чтобы не терять инфу»)
- **Урок:** ретро-слияние дублей было неверным — три «Gripper Close» = три РАЗНЫХ гриппера, при слиянии их per-set описания (модели эспандеров) потерялись. Правильное место правки — сам импорт, не постфактум-чистка.
- **Промпт `program-import` усилен** (структура на выходе ложится в модель без потерь):
  - «ONE MOVEMENT = ONE EXERCISE WITH MANY SETS» — один и тот же снаряд/движение никогда не дробится на несколько «упражнений»; повторные рабочие подходы = подходы одного упражнения. Per-set отличия (нагрузка, модель эспандера, резина, темп, сторона) живут НА ПОДХОДЕ (вес/`notes`), не как новое упражнение и не теряются.
  - **Грипперы:** нагрузка = модель гриппера, не вес → одно упражнение «Стиснення еспандера», по одному подходу на гриппер, weight=null, модель в `notes` подхода. Три гриппера ×5/×2/×4 → одно упр, три подхода, три заметки. (program view уже показывает `s.notes` в строке подхода.)
  - «single»-блок = строго одно упражнение (чтобы не валить несколько движений в один блок, как было в Upper strength).
- **Код-страховка:** при слиянии дубль-упражнения его описание переносится на подход (`setFields(s, ex.notes)`), чтобы инфо не пропадала, даже если модель снова положит её в notes упражнения.
- Гейтвей умеет только `json:true` (строгий JSON), без json-schema — добавлять structured output потребовало бы переписать 3 адаптера; промпт-фикс решает задачу. **Нужен `supabase functions deploy program-import`.**
