> Цель документа: зафиксировать функционал, архитектуру и **жёсткие лимиты костов** до начала разработки.
> _Примечание: документ ведём на русском, но **в самом приложении русского языка нет** — только English + Українська._

## 1. Видение и пользователи

Кросс-платформенное мобильное приложение для записи силовых тренировок (вкл. армрестлинг и гиревой спорт) с аналитикой, ИИ-тренером и интеграцией данных о восстановлении (OURA Ring).

**Пользователи на старте:** 2 (ты + Маша). Архитектура с расчётом на рост до маркетов без переписывания.

**Языки приложения:** **English + Українська** (русского в UI нет). i18n с первого дня.

**Принципы:**
- Невычурный, но качественный модерновый дизайн; графит-форвард палитра.
- Запись тренировки — **быстрая** даже в зале с плохим интернетом → offline-first.
- ИИ и внешние API **под жёсткими лимитами** (день/неделя/месяц + $-kill-switch).
- Бюджет: **до $30/мес за всё** (инфра + ИИ).
- **Воспроизводимость:** всё в облаке + git, поднимается с любого компа (`git clone` → `npm install` → `.env`).

## 2. Технологический стек

| Слой | Выбор | Почему |
|---|---|---|
| Клиент | **React Native + Expo** + TypeScript | Одна кодовая база → Android (Samsung) + iOS (iPhone) + Web. EAS для сборки/публикации. |
| Навигация | Expo Router | Файловая навигация, deep links. |
| UI / стили | **NativeWind** (Tailwind для RN) + дизайн-токены | Быстрая консистентная вёрстка, «модерн без вычурности». |
| i18n | **expo-localization + i18next** | Локали `en` / `uk`, выбор языка на онбординге и в настройках. Без `ru`. |
| Графики | `react-native-gifted-charts` | Объём/тоннаж/прогресс + пост-тренировочная инфографика. |
| Состояние сервера | **TanStack Query** + Supabase client | Кэш, оптимистичные апдейты, синк. |
| Локальное хранилище | **SQLite** (expo-sqlite) + очередь синка | Offline-first запись. |
| Бэкенд | **Supabase (облачный проект)** — Postgres + Auth + Storage + Edge Functions + RLS | Free покрывает 2 юзеров; одна БД с любого компа; SQL удобен для метрик. |
| ИИ | **LLM-шлюз** в Edge Function (адаптеры Claude / OpenAI / Gemini) | Ключи на сервере; маршрутизация по интенту; лимиты и кэш. |
| Здоровье | **OURA Cloud API v2** (OAuth2) через Edge Function | Per-user, опционально (раздел «Подключения»). |
| CI / сборка | **EAS Build + EAS Submit** | Сборка под обе платформы из облака. |

**Платформы:** Android + iOS с первого дня. Web — бонус Expo для тестов / будущей PWA.

## 3. Функциональные модули

### 3.0. Онбординг (первый запуск)
- **Выбор языка** (English / Українська).
- Выбор **единиц измерения kg / lb** тумблером (как на референсе) — сохраняется в профиль, потом меняется в настройках.
- Опц.: рост, текущий вес тела → для аналитики (динамика веса, относительная сила). Тон сообщений — поддерживающий, без осуждения.
- Логин (см. 3.6).

### 3.1. Запись тренировки (ядро)
- Старт сессии (дата, опц. название/сплит).
- **Выбор упражнения через typeahead** — набираешь «pull…» → выпадающий список (поиск по названию + алиасам на обоих языках), избранное, последние.
- Для каждого упражнения — **подходы (sets)**, в каждом:
  - повторы (reps)
  - вес (kg/lb по профилю)
  - **время отдыха** между подходами (таймер с автозапуском после фиксации подхода)
  - **субъективное ощущение** — шкала RPE 1–10 / эмодзи (🟢 легко → 🔴 на пределе)
  - опц. заметка
- «Как в прошлый раз» — дублирование прошлого подхода в один тап.
- Завершение → **пост-тренировочный экран-инфографика** (см. 3.2).
- Всё пишется локально и синкается в фоне.

#### Стартовый каталог упражнений (сид)
Названия и алиасы — на `en` и `uk`.
- **База/классика:** присед, жим лёжа, становая тяга, подтягивания, ассистированные подтягивания, отжимания.
- **Армрестлинг:** натяжка с нижнего блока, кисть «ручкой» на лямках, пронация, имитация движения (борьба).
- **Гири:** рывок, жим, толчок/швунг, болгарский присед, выпады.
- Каждое упражнение: `name_en`, `name_uk`, `muscle_group`, `equipment`, `aliases[]` (для typeahead), `is_global`. Пользователь может добавлять свои.

### 3.2. Аналитика
**Все метрики считаются на бэке (SQL/RPC), детерминированно — ИИ их только озвучивает, не вычисляет.**

- **Пост-тренировочная сводка (сразу после завершения):** инфографика-карточка — «You crushed it!» + тоннаж за тренировку, число подходов/повторов, длительность, побитые PR, сравнение с прошлой такой же тренировкой. Считается локально → мгновенно, без сети.
- **Тоннаж** (Σ вес × повторы) — тренировка / неделя / месяц.
- **Объём** (Σ подходов, Σ повторов) и динамика.
- **Прогресс по упражнению** — рабочие веса во времени, лучший подход.
- **Оценка 1ПМ** (Epley/Brzycki).
- **Частота** и баланс групп мышц.
- **Средний RPE** и тренд (вместе с OURA — индикатор пере-/недотренированности).
- **Личные рекорды (PR)** + уведомление при побитии.
- Стрики/консистентность (мягкая мотивация, без давиловки).

### 3.3. ИИ-тренер
- **LLM-шлюз с маршрутизацией по интенту.** Цепочка: определить интент → выбрать провайдер+модель по таблице → собрать компактный контекст → ответить под лимитами.
  - Маршрутизация по точке входа + опц. дешёвый классификатор (Haiku/правила):
    - короткая поддержка/болтовня → дешёвая модель (Haiku 4.5 / аналог);
    - совет по технике/прогрессии, утренний бриф, недельный разбор → «умная» модель (Sonnet 4.6);
    - глубокий месячный анализ → по запросу (Sonnet/Opus).
  - Адаптеры **Claude / OpenAI / Gemini** за единым интерфейсом → можем точечно ставить провайдер, который дешевле и не уступает.
- **Отвечает на языке пользователя (en/uk).**
- **Контекст компактный:** последние ~8 сообщений + сводка недавних тренировок (числа с бэка) + сегодняшний OURA-снимок. НЕ вся история.
- **Утренний бриф:** при первом заходе утром — короткое сообщение по сну/readiness + настрой на день.
- Тон: позитивный, всегда «за тебя», подбадривает даже когда тяжело; при этом дельный (техника, прогрессия, восстановление), не токсичный позитив.
- Системный промпт кэшируется (prompt caching) → дешевле вход.

### 3.4. Разбор техники по видео (отдельная фаза, премиально-лимитированная)
- Пользователь загружает короткий клип (≤ ~15–20 с) → ИИ даёт разбор техники с подсказками.
- Реализация: нативное видео в дешёвую мультимодалку (кандидат — Gemini Flash) **или** нарезка ключевых кадров → vision-модель.
- **Клип НЕ хранится:** обрабатываем → сохраняем только текстовое саммари (что было на видео + замечания) → клип сразу удаляется.
- **Отдельный жёсткий лимит** (напр. 3 разбора/неделю на юзера) — самая дорогая операция.

### 3.5. Здоровье / OURA (per-user, опционально)
- Раздел **«Подключения»** в профиле — OURA подключается **по желанию, не для всех** (у Маши есть, у тебя может не быть).
- OAuth2 один раз → утренняя подтяжка: **Readiness, качество и длительность сна, HRV, RHR, температура**.
- 1 синк/день/юзер, кэш в `health_snapshots` → дёшево, доступно ИИ-тренеру.
- Абстракция `HealthProvider` — позже Apple Health / Google Fit / Whoop.

### 3.6. Профиль и настройки
- **Аутентификация:** Google / Apple, magic-link на email, и email+пароль (на выбор).
- **Язык (en/uk)**, единицы (kg/lb), тема (тьма/свет), раздел «Подключения».
- Управление каталогом упражнений. Экспорт данных (CSV/JSON).

## 4. Архитектура данных (Postgres / Supabase)

RLS: каждый видит только свои данные.

```
profile            (user_id, display_name, language[en|uk], units, height, bodyweight, prefs)
exercises          (id, owner_id NULL=глоб., name_en, name_uk, muscle_group, equipment, aliases[], is_global)
workouts           (id, user_id, started_at, ended_at, title, notes)
workout_exercises  (id, workout_id, exercise_id, order_index)
sets               (id, workout_exercise_id, reps, weight, rest_sec, rpe, note, completed_at)
personal_records   (id, user_id, exercise_id, type, value, achieved_at)
health_snapshots   (id, user_id, date, readiness, sleep_score, hrv, rhr, temp, raw JSONB)
ai_threads         (id, user_id, created_at)
ai_messages        (id, thread_id, role, content, provider, model, tokens_in, tokens_out, created_at)
ai_usage           (user_id, period_type[day|week|month], period_key, tokens_in, tokens_out, requests, cost_estimate)
video_analyses     (id, user_id, exercise_id, summary, created_at)   -- сам клип не хранится
oura_tokens        (user_id, access_token, refresh_token, expires_at) -- шифрование
```

- Метрики — SQL-вьюхи / RPC (тоннаж, объём, тренды, 1ПМ, PR). Клиент не считает тяжёлое, ИИ не считает вообще.
- Синк: SQLite → очередь → upsert в Supabase по сети.

## 5. Косты и ЖЁСТКИЕ лимиты ИИ

### 5.1. Инфраструктура
- **Supabase Free**: 500 МБ БД, Edge Functions, Storage — для 2 юзеров $0. (Free-проект засыпает после 7 дней простоя; ежедневное использование держит живым.)
- **EAS** — на разработке бесплатно.
- **Публикация (позже):** Google Play $25 разово, Apple $99/год.
- → Вся месячная «горючка» $30 фактически уходит на **ИИ**, и легко держится сильно ниже.

### 5.2. Модели (актуальные цены Claude, за 1М токенов)
| Модель | Вход | Выход | Роль |
|---|---|---|---|
| Haiku 4.5 (`claude-haiku-4-5`) | $1 | $5 | Рутинный чат, подбадривание, классификатор интента |
| Sonnet 4.6 (`claude-sonnet-4-6`) | $3 | $15 | Бриф, разбор, советы по технике/прогрессии |
| Opus 4.8 (`claude-opus-4-8`) | $5 | $25 | Опц. глубокий месячный анализ |
| Gemini Flash / аналог | дёшево | дёшево | Кандидат для видео-разбора (нативное видео) |

Шлюз с адаптерами позволяет A/B-тестить провайдеров и ставить дешевле там, где не теряем в качестве.

### 5.3. Лимиты — многоуровневые (день / неделя / месяц)
Стартовые значения (легко правятся в конфиге):
1. **Сообщения чата:** ~**60/день**, ~**250/неделя**, ~**800/месяц** на юзера.
2. **Токен-бюджет/день:** ~150k input + 30k output на юзера.
3. **Потолок `max_tokens`/ответ:** ~600.
4. **Видео-разборы:** отдельный лимит, напр. **3/неделю** на юзера.
5. **Обрезка контекста** — последние ~8 сообщений + сводка.
6. **Prompt caching** системного промпта.
7. **Глобальный месячный $-kill-switch** — при достижении потолка ИИ отключается до начала месяца (флаг в Edge Function).
8. Ключи Claude/OpenAI/Gemini/OURA — **только на сервере**.

**Оценка:** при этих лимитах и кэше — порядка **$5–15/мес** на двоих. С запасом под $30.

## 6. Дизайн (направление)
- **Палитра: графит-форвард** (тёмный графит как основной), чистый off-white фон в светлой теме, один сдержанный акцент для CTA/PR (финальный оттенок подберём при первых экранах).
- Типографика: гротеск для UI + крупные цифры для весов/таймера/тоннажа.
- Крупные тач-зоны (логировать одной рукой в зале).
- Тёмная тема по умолчанию, светлая опционально.
- Микро-анимации: фиксация подхода, побитие PR, конец таймера отдыха, пост-тренировочная карточка.

## 7. Этапы

**MVP (фаза 1):** онбординг (язык + kg/lb) · auth · сид-каталог упражнений + typeahead · запись тренировки (подходы/повторы/вес/отдых/RPE) · **пост-тренировочная инфографика** · offline-first синк.

**Фаза 2:** Аналитика (тоннаж, объём, прогресс, 1ПМ, PR) + графики.

**Фаза 3:** ИИ-тренер (LLM-шлюз + маршрутизация + лимиты + утренний бриф).

**Фаза 4:** OURA (раздел «Подключения», per-user).

**Фаза 5:** Видео-разбор техники (лимитированный, без хранения клипа).

**Фаза 6:** Полировка дизайна · экспорт · публикация (EAS Submit).
