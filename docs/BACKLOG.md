## 9. Дистрибуция и push: уход с Expo Go на свои билды (решено 2026-07-02)

> Решение Сергея: подвязка на Expo Go не нравится, уходим на собственные билды (EAS).
> Драйверы: (1) remote push невозможен в Expo Go (вырезан с SDK 53) — а нотификации
> критичны для retention («чтобы братики не забывали про приложуньку»); (2) у Маши
> **iPhone** — ей в любом случае нужна апка официальным путём (TestFlight/App Store);
> (3) для андроид-юзеров комьюнити нужен Google Play; (4) пин SDK 54 из-за Expo Go — риск
> (стор обновит Expo Go → старый SDK отвалится). После перехода пин можно пересматривать.

Этапы (по порядку):
1. **Android preview-билд (первый шаг, $0)**: Firebase-проект + `google-services.json`
   (нужны руки Сергея в консоли Firebase) → профиль в `eas.json` → `eas build -p android
   --profile preview` → APK у Сергея вместо Expo Go. `eas update` продолжает доставлять JS
   как сейчас; пересборка — только при смене нативного слоя.
2. **Push-инфраструктура**: таблица `push_tokens` (RLS, свои строки) → клиент
   `getExpoPushTokenAsync` → Edge Function после `review_leaderboard_entry` шлёт в
   Expo Push API (бесплатный). Пуш доходит при убитой апке. Realtime-слой из
   `entry-notifications.tsx` остаётся как мгновенный канал внутри живой апки.
3. **iOS для Маши**: Apple Developer 💰 **$99/год** → iOS-билд → TestFlight; APNs-ключ
   в конфиг push. До этого Маша живёт на Expo Go (без пушей).
4. **Сторы (когда комьюнити пойдёт)**: Google Play 💰 $25 разово (internal testing →
   production), App Store — ревью Apple.
5. **Нотификации-энгейджмент (продумать отдельно, не спамить)**: пуш «тебя подвинули
   в рейтинге» (insert approved-энтри выше твоего результата), дайджест раз в день/неделю,
   напоминание о тренировке. Частоту и опт-аут решить до включения.

Косты: этапы 1–2 — $0 (лимитов бесплатного EAS хватает на наш темп сборок);
дальше $99/год Apple + $25 Google.
